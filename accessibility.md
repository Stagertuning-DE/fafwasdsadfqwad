
# Barrierefreiheit (A11y)

- **Semantik**: Überschriftenhierarchie, Landmark‑Rollen, `<nav>`, `<main>`, `<header>`, `<footer>`.
- **Tastatur**: Mobile‑Menü, FAQ‑Akkordeon und Links sind per Tastatur bedienbar.
- **Kontraste**: Farbschema mit guten Kontrasten in hell/dunkel.
- **Fokus**: Sichtbarer :focus‑Stil (Skip‑Link, Buttons).
- **Bewegung**: `prefers-reduced-motion` wird respektiert.
- **ARIA**: `aria-expanded`, `aria-current`, `aria-label` sinnvoll eingesetzt.
- **Formulare**: Beschriftungen, `aria-describedby`, Validation Placeholder.
- **Responsivität**: Layout bricht auf Mobile sauber um.
